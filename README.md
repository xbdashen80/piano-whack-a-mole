# 钢琴打地鼠 · 小熊滑板

把**电钢琴变成游戏手柄**的网页游戏 / 电钢琴练习工具。面向**零基础**用户：玩着玩着，把琴键位置、手指指法、节奏感顺便练出来。

> 本文件是项目的总说明（介绍 / 使用 / 技术 / 开发历程 / 测试 / 扩展）。
> 项目最初的开工设想见 [`PROJECT.md`](./PROJECT.md)；本文件描述的是**当前实际状态**。

---

## 1. 这是什么 / 怎么玩

屏幕下方画一排琴键，和真琴上的键**位置一一对应**：

- 键的上方随机冒出**目标圈**（就是"地鼠"），圈外一道**倒计时弧线**在缩。
- 在弧线缩没前，用圈里标注的**手指编号**，按下真琴上对应的键。
- **必须按出现的先后顺序敲**：同屏多个地鼠时，只有最早出现的（带白色脉冲环、其余变暗）能命中；越级会提示"按顺序"。
- 按对 → 加分、连击、爆粒子、震屏、悦耳音效。
- 屏幕左侧一只**小熊踩滑板浮在水面**：按对就往上弹，停手就下沉，掉水里 = 失败（拟人化的"血条"）。
- 顶部进度条满 = **通关**，解锁下一关。

### 特色机制

| 机制 | 说明 |
|---|---|
| **节奏同步** | 地鼠卡着背景鼓点冒出；敲在鼓点上（±110ms）给金色特效 + 额外加分。偏拍不会 miss（判定宽松）。 |
| **狂热 Fever** | 屏幕上方"狂热槽"，命中蓄、失误掉、跨关累积；蓄满进入 8 秒 FEVER：分数 ×2、地鼠更密、音乐更炸、全场金光。 |
| **金鼠** | 约 25% 概率的金色地鼠：分数 ×5（叠狂热最高 ×10）、猛蓄狂热、金光大爆。 |
| **炸弹鼠** | 仅后期"进阶·穿指"关出现：**千万别敲**，敲到 = 震撼爆炸 + game over；放着自己消失则无事。 |
| **打击感** | 命中震屏、分数弹字、连击越高画面越"热"（边缘光晕暖→红→品红 + 中央大连击数）、里程碑白闪。 |

### 关卡设计（电钢琴教学法导向）

共 **29 关**，分两段：

- **主体（第 1–20 关）= 五指位**：手不动，最多 5 个键，拇指在最低音、1-2-3-4-5 依次。难度只靠**速度 / 节奏 / 换把位**增长，**不要求穿指、不出炸弹**。
  - C 位（C D E F G）→ 换到 **G 位 / D 位 / A 位**（手整体移过去，仍是五指位）。每换一个新把位，第一关都明显放慢"熟悉把位"。
- **后期（第 21–29 关）= 进阶·穿指音阶**：6→12 个键，需要**拇指穿掌（thumb-under）**。这是真正的进阶技巧，入口很慢、单发，慢慢练；炸弹也只在这一段出现。

> 指法严格按真实钢琴：五指位是 1-5；音阶是 `1 2 3 1 2 3 4`（拇指在 F、C 穿掌），一个八度收尾的主音用 5 指。依据 C.P.E. 巴赫《键盘演奏论》的"短组+长组"传统指法。

## 2. 使用方法

### 硬件 / 浏览器

- 电钢琴（如 Yamaha P-115）用 USB 打印线接电脑，走 **Web MIDI**。
- 浏览器用 **Chrome / Edge**（只有它们支持 Web MIDI）。
- 没插琴时，用电脑键盘 `A S D F G H J K L` 模拟当前把位的琴键（会跟随换把位）。

### 运行（必须本地服务器，不能 `file://`）

项目用原生 ES modules，需要静态服务器：

```bash
# 在项目根目录
python -m http.server 8123
```

然后用 Chrome/Edge 打开 **http://localhost:8123/index.html**

- 首次点"开始"才会初始化音频（浏览器要求用户手势后才能播声）。
- 免鼠标推进：覆盖层上按**空格**，或**连敲两下同一个琴键**，即可开始 / 下一关 / 重来。
- 顶部按钮可独立开关 🥁 鼓点 / 🔊 音效。

## 3. 技术栈

- **纯前端、零构建**：原生 ES modules，无打包工具、无依赖安装。
- **Web MIDI API**：读电钢琴的 note-on 事件。
- **Tone.js**（CDN 引入）：纯合成背景音乐（鼓机 + 贝斯 + 旋律 + pad）与所有音效。**音色路线 = 纯合成，不引外部采样**（保持单页自包含、零额外资源）。
- **Canvas 2D**：画键盘、目标圈、小熊、粒子、水面、所有特效。
- **localStorage**：存档（最高分、已解锁关卡）。
- 主循环 `requestAnimationFrame`。

## 4. 代码架构

`src/` 下是单向无环的 ES 模块；模块间的**反向调用**一律走 `events.js` 事件总线（如 `midi → 'press' → game`），以此打破循环依赖。

```
index.html         入口，引 Tone.js(CDN) + src/main.js
src/
  main.js          装配：注册监听、初始化、启动主循环
  events.js        极简事件总线 on/emit（打破反向依赖）
  state.js         共享状态中枢：画布/视口/存档/game 状态/键盘几何/换把位
  levels.js        纯数据：白键表、29 关数据、指法 fingerFor、音乐谱 MUSIC、配色
  midi.js          输入：Web MIDI + 键盘兜底 → 统一 'press' 事件
  audio.js         Tone.js 封装：背景乐主控链 + 鼓机/编排引擎 + 各类音效
  game.js          主循环 tick、状态机、判定、计分、过关/失败、节拍生成、狂热
  render.js        Canvas 绘制：键盘、目标圈、粒子、震屏、连击热度、爆炸、叠加层
  bear.js          小熊 + 滑板 + 水面的绘制与物理
  ui.js            HUD、进度条、Toast、结算页、选关网格、开始界面预览
  diagnostics.js   诊断仪表：全局错误兜底 + 生成停摆看门狗（卡死可定位）
```

### 几个关键设计点

- **事件总线解耦**：输入、UI、物理都通过 `emit` 把事件回流到 `game.js`，模块依赖图保持单向。
- **音频/游戏双时钟桥接**：音乐跑在 Tone 的音频时钟、判定用 `performance.now()`。鼓机循环用 `Tone.Draw.schedule(cb, time)` 在"音频时间真正到达"的那一视觉帧发拍，回调里的 `performance.now()` ≈ 可听见的鼠点 → 地鼠生成与命中判定都能真正"踩"在鼓点上。拍源停摆（关 BGM/音频失败）时 `tick` 退回墙钟生成，保证不卡。
- **换把位 `off`**：一关的活动键 = `WHITE.slice(off, off+keys)`，而非固定从 C。`activeMidis(L)` 是唯一出口，`state.layoutKeys` / `ui.renderKbPreview` / `midi` 键盘兜底都用它。
- **副作用绝不阻断状态机**：所有音效用 `safe()` 包裹、异常只 `logErr` 不冒泡（P0 卡死的教训，见下）。

```bash
for f in test/*.mjs; do [ "$(basename $f)" = _tone-stub.mjs ] || node "$f"; done
```

现有 10 个测试（含桩文件）：

| 文件 | 覆盖 |
|---|---|
| `smoke-import.mjs` | 全模块能正常 import、装配不报错 |
| `p0-levelclear.mjs` | **P0 回归**：音效抛错时过关弹窗仍正常、流程不中断（故意用真 Tone 触发错误） |

| `beat-sync.mjs` | 踩点加分 / 偏拍不加分 / 命中不被挡 / 'beat' 刷新锚点 |

## 7. 调参 / 扩展指引

大多数手感参数集中、好调：

- **关卡难度**：`src/levels.js` 的 `LEVELS`（每关 `keys/off/spawn/ttl/maxActive/goal/sink/bounce/music/t`）。`off` 控制换把位。
- **指法**：`src/levels.js` 的 `fingerFor(idx, keys)`。
- **背景音乐 / 编排**：`src/audio.js` 的 `MUSIC`（在 levels.js）与 `drumLoop`（编排强度、chorus 抬八度、fill、侧链泵等旋钮都在这）。
- **炸弹**：`src/game.js` 的 `BOMB_FROM_LEVEL`（起始关）、`spawnMole` 里的概率、`press` 炸弹分支的爆炸强度（`hitStop` / `shake` / `bombFlash`）。
- **狂热**：`game.js` 的 `addFever` 蓄量、`startFever` 时长、失误 `*0.6`。
- **打击感**：`game.js` 命中处的 `shake` / `comboFlash`，`render.js` 的 `drawOverlays` 与 `fx`/`fxBoom`。

**还没做、可继续的方向**：快鼠、道具 buff（护盾/双倍/减速）、双手或左手镜像模式、黑键 / 和弦、把抽象"目标圈"做成真地鼠图案、进阶轨做成独立选关分区。

## 8. 设计原则（始终遵守）

- **零基础友好**：起点极低、提示直白、不假设懂乐理。
- **教学法正确**：指法、把位、穿指的引入符合真实钢琴技法，循序渐进。
- **游戏性第一**：即时反馈、紧张感、明确目标与成长解锁。
- **声音饱满**：纯合成也要有"街机存在感"。
- **改动增量化、可回退**：每步小提交，配回归测试防止反复。
- **副作用不阻断主流程**：音效等副作用异常只记录、绝不冒泡进状态机。
